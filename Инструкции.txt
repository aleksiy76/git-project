git init - определить папку как репозиторий git 
git config --global user.name "aleksey" - установка имени пользователя
git config --global user.email aleksiy76@yandex.ru - установка e-mail пользователя

Если создать текстовый файл и сохрани ег с расширением .gitignore без имени, то в нем можно указывать папки которые необходимо игнорировать
Например 
#folder logs
logs/

#txt-files
docs/*.txt - игнор в папке docs файлов с расширением txt
------------------

git status - показ текущего статуса проекта
cd docs - переход в папку docs глазами git
git status --untracked-files=normal - показать файлы которые еще не в git (normal или all)

git add . - добавить все файлы
git rm --cashed license.php удалить файл license.php 
git rm -r --cached Git/ - удалить всю папку

git commit -a -m"init" (-a -все изменненые файлы  -m"init" - это месседж - просто говорим, что это init )

Если забыть поставить -m коммент то откроется окно с просьбой добавить коммент нужно его добавить потом нажать esc двоеточие и x (икс) это значит сохранить и выйти. Чтобы открывался виндовый редактор - можно посмотреть 7 урок 4 минута (Свой редактор для комментраиев)




git log - посмотреть историю изменений
git log --pretty=format:"%h - %an, %ar : %s" - компактный вид 
git log --since=2.weeks - показать коммиты за 2 недели
git log -p -2 - показ коммитов и подробными изменениями  ( -2 это два последних коммита)


Создание новой ветки:
git checkout -b new_f  создаем новую ветку и сразу на нее переключаемся
git branch new_f2 создаем новую ветку, но не переключаемся на неё
git checkout new_f - переключение на ветку new_f 


Мерж:

git config --global merge.tool kdiff3 - устанока утилиты в которой будем мержить вручную
Скачать ее можно здесь:
http://sourceforge.net/projects/kdiff3/files/
После установке в баше установить
git config --global mergetool.kdiff3.cmd '"C:\\Program Files\\KDiff3\\kdiff3" $BASE $LOCAL $REMOTE -o $MERGED'
